PocketBase MCP CLI

Run MCP in minutes, keep every client in sync.

PocketMCP is the direct path to serve a PocketBase admin MCP endpoint over stdio and roll out config to your MCP clients with one clean install flow.

Installers fetch the latest release by default. Set VERSION to pin a specific tag.

Install PocketMCP

Installed binary names: pocketmcp (Linux/macOS) and pocketmcp.exe (Windows).

curl -fsSL https://raw.githubusercontent.com/mreyeswilson/pocketmcp/main/install.sh | bash
powershell -c "irm https://raw.githubusercontent.com/mreyeswilson/pocketmcp/main/install.ps1 | iex"

Focused command surface

Use dedicated `serve` and `install` commands designed for predictable MCP setup instead of generic script sprawl.

Client config automation

Bootstrap MCP-compatible clients quickly with generated config that stays consistent across environments.

Release-ready workflow

Ship reproducible installs from tagged releases so teams can pin versions and avoid drift.

Typical setup flow

01

Install binary

Run the install script for your shell and keep PocketMCP available on your PATH.

02

Run MCP server

Start an authenticated stdio MCP session against your PocketBase admin endpoint.

03

Install client config

Apply generated settings for one or all supported clients to unify local tooling.

Core commands

pocketmcp serve \
  --url http://127.0.0.1:8090 \
  --email admin@example.com \
  --password 'your_password' \
  --timeout-ms 15000

pocketmcp install \
  --client all \
  --url http://127.0.0.1:8090 \
  --email admin@example.com \
  --password 'your_password'

# Windows binary filename:
pocketmcp.exe --help

When to use PocketMCP

  • You need a repeatable admin MCP setup for local automation.
  • Your team wants one install flow for Linux, macOS, and Windows.
  • You want release-tagged binaries for stable CI and onboarding docs.
